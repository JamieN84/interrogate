# Vertical Slice Plan

## Objective

Define the minimum shippable vertical slice boundary.

## Definition Of Done (Playable Means)

- Player can start the game.
- Core loop runs.
- Correct and incorrect input are evaluated.
- Score increments.
- Failure triggers reset.
- Basic visual feedback is present.
- No crashes during normal play.

## What Is Explicitly Excluded

- Any feature not listed in the Definition Of Done section.
- Campaign mode and multi-case narrative progression.
- Endless mode and daily challenge systems.
- Leaderboard services and backend integration.
- Save systems (local or cloud), account systems, and profile sync.
- Advanced difficulty tuning, progression systems, and unlock economies.
- Procedural generation and content authoring tools.
- Localization and regionalized content.
- High-fidelity audio/visual polish work.
- Controller/haptics tuning and platform-specific polish.

## What Polish Is Intentionally Ignored

- Final art direction, cinematic transitions, and branding polish.
- Advanced animation polish beyond functional readability.
- Audio mix/mastering polish.
- Extended accessibility enhancements beyond basic readability.
- Copy polish and narrative text refinement.
- Performance optimization beyond crash-free normal play.

## Exit Criteria

- All Definition Of Done items are true.
- No blocker crash is observed in normal play testing.
