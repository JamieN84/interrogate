# interrogate

## Documentation Structure

- `docs/product.md`: product-level direction and creative intent
- `specs/*.spec.md`: system-level implementation rules
- `plans/*.plan.md`: delivery scope and milestone plans
- `decisions/*.md`: ADR-style architectural and scope decisions
- `roadmap/backlog.md`: future docs/features to create later

## Templates

Use templates from `docs/templates/` when creating new documents:

- `docs/templates/product.template.md`
- `docs/templates/spec.template.md`
- `docs/templates/plan.template.md`
- `docs/templates/adr.template.md`
- `docs/templates/backlog.template.md`
